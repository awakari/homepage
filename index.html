About

Awakari is a service that combines publish/subscribe capability with real-time search.

How it works

Awakari is capable to consume event streams from any source.

A user expresses their interest as a subscription with a certain search criteria.

For every incoming event, Awakari does the following:

  • Resolves matching subscriptions
  • Notifies the online user immediately

A user is also able to publish own events to Awakari.

    Comparison

    Event Driven

    • Push notifications reduce the latency.
    • Avoids repeated dry polls cost. ⧉

    Inverse Search

    • Finds queries by incoming event.
    • More-cost efficient for large number of queries.

    How does Inverse Search reduce costs?

    Given: K input events per second, N registered queries.

    In conventional stream data processing, every incoming event is processed by every registered query. Hence, the resulting load may be estimated as K * N. For example, AWS Kinesis Data Analytics charges per KPU ⧉ that is in direct ratio to the load.

    In case of Inverse Search, the search is executed once for every incoming event. The Inverse Search itself uses the index, hence every search operation complexity is O(log(N)). The resulting load is K * log(N).
