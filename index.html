About

Conventional search is used to find an important information by a query. The problem is reverse in the modern world of real-time message streams. It's necessary to filter and consume messages matching a query on the fly. Existing messaging solutions offer only "channels" or "topics" for this. Nevertheless, these tend to contain tons of irrelevant information.

Awakari comes to solve this problem by using flexible subscriptions instead of channels.

  • Cloud Native
  • Instant Delivery
  • Relevant Only Info

Features

  • Instant Delivery

    • Get notified about the new messages in the real time.

  • Relevant Messages

    • Subscribe using custom message matching conditions.

  • Arbitrary Metadata

    • Specify custom keys to match: category, location, ...

  • Pattern Matching

    • For example *el?ow will match: yellow, elbow, ...

  • Composite Conditions

    • Groups of nested conditions and groups

  • Logic Functions

    • And, Or, Xor, Not

    Use Cases

  • Get Notified

    • Subscribe for important info and receive the relevant information in the real time.

  • Advertise

    • Publish own information and everyone who interested in this will receive your message.

    Quick Start

    The instructions below apply to the cloud instances only. To start using self-hosted instance, follow this link ⧉.

    1. Install grpcurl
    2. Get a cloud client certificate
    3. Get the Cloud Events protobuf file and save it in the current directory
    4. Check access: 
      grpcurl \
  -key shared-demo.awakari.cloud.key \
  -cert shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d '{"subj": 1}' \
  demo.awakari.cloud:443 \
  awakari.api.limits.Service/Get
    5. Create a subscription: 
      grpcurl \
  -key shared-demo.awakari.cloud.key \
  -cert shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d @ \
  demo.awakari.cloud:443 \
  awakari.subscriptions.Service/Create
      Insert the subscription input data and press ^D: 
      {
   "md": {
      "description": "John Doe's test subscription 0",
      "enabled": true
   },
   "cond": {
      "not": false,
      "ktc": {
        "key": "subject",
        "pattern": "Kenobi",
        "partial": true
      }
    }
}
      The successful response looks like: 
      {
  "id": "e8ed2e43-8e55-4765-8183-c743fb2c3612"
}
      Note the created subscription id and use it further to read the messages.
    6. Start reading the messages by the subscription: 
      grpcurl \
  -proto cloudevent.proto \
  -max-time 86400 \
  -key shared-demo.awakari.cloud.key \
  -cert shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d @ \
  demo.awakari.cloud:443 \
  awakari.reader.Service/Read
      Specify the subscription id in the request and press Enter: 
      {"start": {"batchSize": 1, "subId": "e8ed2e43-8e55-4765-8183-c743fb2c3612"}}
    7. Switch to another shell/window to publish a message: 
      grpcurl \
  -proto cloudevent.proto \
  -key shared-demo.awakari.cloud.key \
  -cert shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d @ \
  demo.awakari.cloud:443 \
  awakari.writer.Service/SubmitMessages
      Then send the request with a message payload: 
      {
   "msgs": [
      {
         "id": "3426d090-1b8a-4a09-ac9c-41f2de24d5ac",
         "type": "example.type",
         "source": "example/uri",
         "spec_version": "1.0",
         "attributes": {
            "subject": {
               "ce_string": "Obi-Wan Kenobi"
            },
            "time": {
               "ce_timestamp": "1985-04-12T23:20:50.52Z"
            }
         },
         "text_data": "I felt a great disturbance in the force"
      }
   ]
}
      The successful response looks like: 
      {
  "ackCount": 1
}
    8. Switch back to the window/shell where reading was started and observe the message is received
    9. Acknowledge the received message by typing: 
      {"ack": { "count": 1}}
    10. That's it! Delete the subscription to clean up: 
      grpcurl \
  -key shared-demo.awakari.cloud.key \
  -cert shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d '{"id": "e8ed2e43-8e55-4765-8183-c743fb2c3612"}'
  demo.awakari.cloud:443 \
  awakari.subscriptions.Service/Delete
