About

In the modern world of information streams its getting harder to find what is important timely. Conventional search engines, social networks and messengers doesn't solve this problem.

Awakari offers comprehensive subscriptions to catch the relevant messages among everything. It allows users to publish their messages to the common stream. Other users decide, what they want to receive.

  • Instant Notifications
  • Relevant Only Messages

Awakari works with:

  • Streams of messages in the Cloud Events ⧉ format.
  • Subscriptions containing sets of matching conditions.

Features

  • Instant Notifications

    • Get notified about the new messages timely.

  • Relevant Only Messages

    • Subscribe using custom message matching conditions.

  • Arbitrary Attributes

    • Specify custom keys to match: category, location, ...

  • Pattern Matching

    • For example *el?ow will match: yellow, elbow, ...

  • Composite Conditions

    • Groups of nested conditions and groups

  • Logic Functions

    • And, Or, Xor, Not

    API

    Awakari provides the following core gRPC functions to work:

    • Manage Subscriptions
    • Message Receiving Streams
    • Message Publishing Streams

    Quick Start

    The instructions below apply to the cloud instances only. To start using self-hosted instance, follow this link ⧉.

    Preparation Steps:

    1. Install grpcurl
    2. Download the CA file for the demo instance and save it in the current directory
    3. Download the shared client certificate
      4. and save it to the current directory
    4. Download the shared client key
      5. and save it to the current directory
    5. Download the Cloud Events proto file and save it in the current directory
    6. Download the Reader service proto file and save it in the current directory
    7. Download the Writer service proto file and save it in the current directory

    Basic Usage Walkthrough:

    1. Check access: 
      grpcurl \
  -key client-shared-demo.awakari.cloud.key \
  -cert client-shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d '{"subj": 2}' \
  demo.awakari.cloud:443 \
  awakari.api.limits.Service/Get
      A successful response looks like: 
      {
  "count": "123"
}
      That means that the count limit for the subject 2 (subscriptions) is 123. There's also a subject 1 meaning the count limit for the messages those can be published. The missing attribute "userId" in the response means that the limit is group-wide. A group-wide limit means it's shared with other users using the same certificate.
    2. Next, check the current group-wide usage (skipping the "X-Awakari-User-Id" header): 
      grpcurl \
  -key client-shared-demo.awakari.cloud.key \
  -cert client-shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -d '{"subj": 2}' \
  demo.awakari.cloud:443 \
  awakari.api.permits.Service/GetUsage
      A successful response looks like: 
      {
  "count": "1",
  "countTotal": "1",
  "since": "2023-06-18T14:39:18.087Z"
}
      Meaning there's 1 subscription have been spent. Hence, the remaining count of subscriptions those can be created is 123 - 1 = 122.
    3. Create a subscription: 
      grpcurl \
  -key client-shared-demo.awakari.cloud.key \
  -cert client-shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d @ \
  demo.awakari.cloud:443 \
  awakari.subscriptions.Service/Create
      Insert the subscription input data and press ^D: 
      {
   "md": {
      "description": "John Doe's test subscription 0",
      "enabled": true
   },
   "cond": {
      "not": false,
      "ktc": {
        "key": "subject",
        "pattern": "Kenobi",
        "partial": true
      }
    }
}
      A successful response looks like: 
      {
  "id": "547857e3-adfc-48a5-a49e-110cfdedbaab"
}
      Note the created subscription id and use it further to read the messages.
    4. Start reader stream: 
      grpcurl \
  -proto reader.proto \
  -max-time 86400 \
  -key client-shared-demo.awakari.cloud.key \
  -cert client-shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d @ \
  demo.awakari.cloud:443 \
  awakari.reader.Service/Read
      Specify the subscription id in the request and press Enter: 
      {"start": {"batchSize": 1, "subId": "547857e3-adfc-48a5-a49e-110cfdedbaab"}}
    5. Switch to another shell/window and start the writer stream: 
      grpcurl \
  -proto writer.proto \
  -key client-shared-demo.awakari.cloud.key \
  -cert client-shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d @ \
  demo.awakari.cloud:443 \
  awakari.writer.Service/SubmitMessages
      Then publish a message: 
      {
   "msgs": [
      {
         "id": "3426d090-1b8a-4a09-ac9c-41f2de24d5ac",
         "type": "example.type",
         "source": "example/uri",
         "spec_version": "1.0",
         "attributes": {
            "subject": {
               "ce_string": "Obi-Wan Kenobi"
            },
            "time": {
               "ce_timestamp": "1985-04-12T23:20:50.52Z"
            }
         },
         "text_data": "I felt a great disturbance in the force"
      }
   ]
}
      A successful response looks like: 
      {
  "ackCount": 1
}
      After this, you can close the writer stream by pressing ^C or leave it open to publish any other messages later.
    6. Switch back to the reader stream window/shell and observe the message is received
    7. Acknowledge the received message by typing: 
      {"ack": { "count": 1}}
      After this, you can close the reader stream by pressing ^C or leave it open to read any other messages later.
    8. That's it! Delete the subscription to clean up: 
      grpcurl \
  -key client-shared-demo.awakari.cloud.key \
  -cert client-shared-demo.awakari.cloud.crt \
  -cacert ca-demo.awakari.cloud.crt \
  -H 'X-Awakari-User-Id: john.doe' \
  -d '{"id": "e8ed2e43-8e55-4765-8183-c743fb2c3612"}'
  demo.awakari.cloud:443 \
  awakari.subscriptions.Service/Delete
